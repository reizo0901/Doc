じゃんけんで考える作業工程とプログラム

【要件】
コンソール上でコンピュータと勝負できる「じゃんけん」ゲームを作成する。
「じゃんけん」とは、３種類の手の型で「３すくみ」の関係を構成し、その相性により
勝敗を決める遊戯である。
手の型には石を模した「ぐー」、ハサミを模した「ちょき」、紙を模した「ぱー」が存在し、
石はハサミに勝ち、ハサミは紙に勝ち、紙は石に勝つといった「３すくみ」で構成される。

プレイヤーが選択した手の型とコンピュータがランダムで選択した手の型を判定し、
勝敗を決めます。

追加要件①
　繰り返し「じゃんけん」が出来るようにする。

追加要件②

追加要件③
　勝敗結果を表示する。

追加要件④
　コンピュータのレベルを変更出来るようにする。

・まずは処理の流れを日本語を使ってフローチャートにしてみましょう。

　※フローチャートとは
　　作業や業務、処理の流れについて記号を使って分かり易くしたものです。
　　それぞれの記号には意味があったり、矢印の書き方などのルールも存在します。
　　会社やお客様毎に書き方のルールが違う場合もありますので都度確認が必要です。

とりあえず以下の図形を利用します。
終端記号(開始と終了)
ループ記号(ループ開始と終了)
判断記号(条件式)
手動入力記号
処理記号

・フローチャートができたら、日本語で書いた部分をプログラムに置き換えてみましょう。

【ヒント】

●ランダムな数を生成するには？

　コンピュータが出す手はrandam関数を利用します。
　randam変数は言語によって発生する数値の範囲が異なります。

　VBSとVBAの場合
　rnd
　発生数値：0～1未満

　バッチの場合
　%random%
　発生数値：0～32767

●条件で分岐するには

　VBSの場合

　If 条件式 Then
　　処理1
　End If
　
　If 条件式 Then
　　処理1
　ElseIf 条件式 Then
　　処理2
　Else
　　処理3
　End If

　バッチの場合
　
　If 条件式 (
　　処理1
　)
　
　If 条件式 (
　　処理1
　) Else If 条件式 (
　　処理2
　) Else (
　　処理3
　)

If文以外にもCase文が存在していますが、とりあえずIf文でトライしてみましょう。

●ループ(繰り返し処理)するには

　VBSの場合
　
　(1)決まった回数ループする場合は
　
　For 変数 = 開始値 To 終了値 (Step 増減数)
　　処理1
　Next
　
　※ループを途中で抜ける場合は処理の部分に"Exit For"を書く。
　
　(2)決まった条件を満たしている時だけループする場合は
　
　前条件のループ
　
　While 条件
　	処理1
　Wend
　
　Do While 条件式
　　処理1
　Loop
　
　後条件のループ
　
　Do
　　処理1
　Doop While 条件式
　
　※ループを途中で抜ける場合は処理の部分に"Exit Do"を書く。
　※前条件と後条件のループの違いは、前条件の場合は、条件が満たされていなければ
　　ループに入らず１回も処理されません。
　　後条件の場合は、必ず１回は処理されます。
　
　バッチの場合
　
　For (オプション) %%変数 In (セット) Do (
　　処理1
　)
